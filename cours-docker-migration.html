C'est quoi Docker ?

Docker permet d'empaqueter une application avec tout ce dont elle a besoin pour fonctionner dans des "containers". Le site fonctionne de la même façon partout !

Le Problème sans Docker

Sans Docker
  • ❌ "Ça marche sur mon PC mais pas chez toi"
  • ❌ Installer PHP, MySQL, Apache manuellement
  • ❌ Configurer tout un par un
  • ❌ Versions différentes = bugs
Avec Docker
  • ✅ "Ça marche partout pareil"
  • ✅ Une seule commande pour tout
  • ✅ Configuration automatique
  • ✅ Mêmes versions garanties

Analogie Simple

Docker = Container maritime 🚢

Imagine un container maritime : peu importe ce qu'il y a dedans, il peut être transporté par bateau, camion ou train.

Docker c'est pareil : ton application est dans un "container" qui fonctionne sur Windows, Linux, Mac... sans rien changer !

À retenir

  • Docker = Empaqueter une application avec tout ce qu'il lui faut
  • Container = Boîte isolée qui contient l'application
  • "Ça marche sur ma machine" → "Ça marche partout"

Images & Containers

Comprendre la différence entre une Image et un Container, c'est comprendre Docker.

Image vs Container

Image 📀 Container 📦
C'est un modèle (template) C'est une instance en cours d'exécution
Comme un fichier ISO de Windows Comme Windows installé et lancé
Ne change jamais (read-only) Peut être modifié, arrêté, relancé
docker images docker ps
Analogie

Image = Recette de cuisine 📖
Container = Le plat cuisiné 🍝

À partir d'une même recette, tu peux faire plusieurs plats identiques !

Images Officielles utilisées

Image Description Utilisation
php:8.3-apache PHP 8.3 + Apache Serveur web PHP
mysql:8.0 MySQL 8.0 Base de données
phpmyadmin Interface graphique MySQL Administrer la BDD

À retenir

  • Image = Modèle figé (recette)
  • Container = Instance en cours d'exécution (plat)
  • Une image peut créer plusieurs containers

Le Dockerfile

Le Dockerfile est la "recette" pour construire ta propre image Docker personnalisée.

Le Dockerfile du projet BTS CIEL

Dockerfile 
# 1. Image de base (point de départ)
FROM php:8.3-apache

# 2. Installer des extensions PHP nécessaires
RUN apt-get update && apt-get install -y \
    libpng-dev \
    libzip-dev \
    && docker-php-ext-install pdo_mysql gd zip

# 3. Activer le module Apache rewrite (pour Laravel)
RUN a2enmod rewrite

# 4. Configurer Apache pour pointer vers /public
RUN sed -i 's|/var/www/html|/var/www/html/public|g' \
    /etc/apache2/sites-available/000-default.conf

# 5. Installer Composer (gestionnaire de dépendances PHP)
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# 6. Définir le dossier de travail
WORKDIR /var/www/html

# 7. Copier le projet dans le container
COPY . .

# 8. Installer les dépendances PHP
RUN composer install --no-dev --optimize-autoloader

# 9. Donner les permissions à Apache
RUN chown -R www-data:www-data /var/www/html/storage

# 10. Exposer le port 80
EXPOSE 80

Explication des instructions

Instruction Explication
FROM Image de base sur laquelle on construit
RUN Exécute une commande pendant la construction
COPY Copie des fichiers de ton PC vers l'image
WORKDIR Définit le dossier de travail par défaut
EXPOSE Indique quel port le container utilise

À retenir

  • Dockerfile = Recette pour créer une image
  • FROM = Point de départ
  • RUN = Commandes d'installation
  • COPY = Copier tes fichiers

Docker Compose

Docker Compose permet de gérer plusieurs containers ensemble avec un seul fichier.

Pourquoi Docker Compose ?

Sans Docker Compose

docker run -d --name app ...
docker run -d --name db ...
docker run -d --name pma ...
docker network create ...
docker network connect ...

😵 Trop de commandes !

Avec Docker Compose

docker-compose up -d

😎 Une seule commande !

Le fichier docker-compose.yml

docker-compose.yml 
services:
  # Container 1 : Application Laravel
  app:
    build: .                          # Construit depuis le Dockerfile
    container_name: bts-ciel-app
    ports:
      - "80:80"                        # Port local:port container
    volumes:
      - ./public/img:/var/www/html/public/img
    depends_on:
      db:
        condition: service_healthy

  # Container 2 : Base de données MySQL
  db:
    image: mysql:8.0
    container_name: bts-ciel-db
    environment:
      MYSQL_ROOT_PASSWORD: root_secret
      MYSQL_DATABASE: bts_ciel
      MYSQL_USER: bts_user
      MYSQL_PASSWORD: bts_secret_2024
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Container 3 : phpMyAdmin
  phpmyadmin:
    image: phpmyadmin:latest
    container_name: bts-ciel-phpmyadmin
    ports:
      - "8080:80"
    environment:
      PMA_HOST: db
    depends_on:
      - db

volumes:
  mysql_data:

Commandes Docker Compose

Commande Description
docker-compose up -d Lancer tous les containers
docker-compose up -d --build Lancer + reconstruire les images
docker-compose down Arrêter tous les containers
docker-compose ps Voir l'état des containers
docker-compose logs Voir les logs
docker-compose exec app bash Entrer dans le container app

À retenir

  • docker-compose.yml = Configuration de tous les containers
  • docker-compose up -d = Tout lancer
  • docker-compose down = Tout arrêter

Exporter la Base de Données

Avant de migrer, il faut exporter les données de MySQL dans un fichier .sql.

La commande mysqldump

Terminal (PC source) 
# Se placer dans le dossier du projet
cd C:\wamp64\www\bts-ciel-laravel

# Exporter la base de données
docker-compose exec db mysqldump --no-tablespaces -u bts_user -pbts_secret_2024 bts_ciel > backup.sql

Décortiquons cette commande :

Partie Signification
docker-compose exec db Exécute une commande dans le container "db"
mysqldump Outil pour exporter une BDD MySQL
-u bts_user Utilisateur MySQL
-pbts_secret_2024 Mot de passe (collé au -p !)
bts_ciel Nom de la base de données
> backup.sql Redirige vers un fichier

À retenir

  • mysqldump = Outil pour exporter une BDD
  • Le fichier .sql contient la structure + les données
  • Toujours vérifier que le fichier n'est pas vide !

Transfert des Fichiers

Comment copier ton projet vers le nouveau PC/serveur.

Via Clé USB / Disque externe

Sur le PC source (Windows)

Copie tout le dossier bts-ciel-laravel sur ta clé USB / disque externe

Sur le PC destination (Linux)
Terminal Linux 
# Copier depuis le disque externe
sudo cp -r "/media/user/MON_DISQUE/bts-ciel-laravel" /var/www/

# Donner les permissions
sudo chown -R $USER:$USER /var/www/bts-ciel-laravel

Ce qu'il faut copier

Élément Fichier/Dossier Contient
📁 Code source Tout le dossier PHP, Blade, routes...
🐬 Base de données backup.sql Actualités, fichiers, admins
🖼️ Images public/img/ Photos, fichiers uploadés
⚙️ Configuration .env, docker-compose.yml Paramètres

À retenir

  • Copier tout le dossier du projet
  • Ne pas oublier backup.sql !
  • Donner les permissions avec chown

Déploiement sur Linux

Les étapes pour lancer ton application sur le nouveau PC Linux.

Étapes complètes

Installer Docker
Terminal Linux 
sudo apt update
sudo apt install docker.io docker-compose -y
sudo systemctl start docker
Copier le projet depuis le disque externe
Terminal 
sudo cp -r "/media/user/MON_DISQUE/bts-ciel-laravel" /var/www/
sudo chown -R $USER:$USER /var/www/bts-ciel-laravel
Aller dans le dossier
Terminal 
cd /var/www/bts-ciel-laravel
Lancer Docker Compose
Terminal 
sudo docker-compose up -d --build

⏳ Attends 2-5 minutes...

Vérifier que ça tourne
Terminal 
sudo docker-compose ps

# Tu dois voir 3 containers "Up"
Attendre 30 secondes
Terminal 
sleep 30
Importer la base de données
Terminal 
sudo docker-compose exec -T db mysql -u bts_user -pbts_secret_2024 bts_ciel < backup.sql
Tester ! 🎉
http://localhost → Site BTS CIEL
http://localhost/admin/login → Dashboard admin
http://localhost:8080 → phpMyAdmin

Problèmes courants

Erreur Solution
"Permission denied" Ajoute sudo devant la commande
"docker-compose: command not found" Essaie docker compose (sans tiret)
"Port 80 already in use" sudo systemctl stop apache2

À retenir

  • Installer Docker en premier
  • docker-compose up -d --build pour lancer
  • Attendre 30 sec avant d'importer la BDD
