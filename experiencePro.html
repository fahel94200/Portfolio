Site Web BTS CIEL

Refonte complète d'un site pédagogique avec Laravel, Dashboard administrateur sécurisé et déploiement Docker

Voir l'aperçu du site
Lycée Christophe Colomb, Sucy-en-Brie
Janvier - Février 2026 (6 semaines)
Développeur Full Stack

Contexte du Projet

Le Lycée Christophe Colomb disposait d'un site web statique pour la section BTS CIEL (Cybersécurité, Informatique et réseaux, Électronique). Ce site nécessitait une refonte complète pour moderniser son architecture et faciliter sa maintenance.

Ma mission consistait à transformer ce site en une application web dynamique avec un système de gestion de contenu permettant aux enseignants de mettre à jour facilement les ressources pédagogiques.

Objectifs

  • Migrer l'ensemble des pages HTML statiques vers le framework Laravel
  • Développer un dashboard administrateur complet et sécurisé
  • Implémenter un système de gestion des actualités (CRUD)
  • Créer un gestionnaire de fichiers pour les cours et projets
  • Containeriser l'application avec Docker pour le déploiement

Analyse & Conception

Modélisation UML avant le développement

Diagramme de Cas d'Utilisation

Avant de commencer le développement du site en Laravel, j'ai réalisé un diagramme de cas d'utilisation (Use Case) afin de formaliser l'ensemble des fonctionnalités prévues et de structurer ma réflexion.

Ce diagramme m'a permis de présenter clairement mes idées à mon tuteur de stage, qui a pu suivre mon raisonnement et valider chaque fonctionnalité avant que je ne commence à coder. C'est une étape essentielle pour s'assurer que le développeur et le commanditaire partagent la même vision du projet.

Acteurs identifiés
  • Client (visiteur) — Consulter les pages BTS, filtrer les catégories, consulter et télécharger les ressources pédagogiques
  • Admin (enseignant) — S'authentifier, administrer le site, gérer les fichiers et les actualités, créer des comptes admin, modifier son mot de passe
Relations modélisées
  • « include » — L'administration du site inclut obligatoirement l'authentification et la déconnexion
  • « extends » — Les actions spécifiques (créer/supprimer actualités, téléverser/supprimer fichiers) étendent les cas d'utilisation principaux

Mes Réalisations

Les différentes étapes du développement

Migration vers Laravel

Transformation complète du site statique en application MVC avec le framework Laravel et le moteur de templates Blade.

  • Migration de 35+ pages HTML vers Blade
  • Création d'un layout principal réutilisable
  • Mise en place du routage Laravel
  • Gestion des assets avec {{ asset() }}
  • Footer moderne et responsive

Dashboard Administrateur

Développement d'une interface d'administration complète pour la gestion du contenu du site.

  • Interface intuitive et responsive
  • Gestion des actualités (Créer, Modifier, Supprimer)
  • Gestionnaire de fichiers (Upload, Renommer, Supprimer)
  • Création de nouveaux administrateurs
  • Modification du mot de passe

Sécurité Renforcée

Mise en place de mesures de sécurité robustes pour protéger l'accès administrateur.

  • Hashage des mots de passe avec bcrypt
  • Session timeout après 10 min d'inactivité
  • Middleware d'authentification personnalisé
  • Protection CSRF sur les formulaires
Détails

Containerisation Docker

Configuration complète pour le déploiement de l'application en environnement de production.

  • Dockerfile optimisé (PHP 8.3 + Apache)
  • docker-compose avec 3 services
  • Container MySQL 8.0 avec persistance
  • phpMyAdmin pour la gestion BDD
  • Scripts de déploiement automatisés
