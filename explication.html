Présentation du Projet

Bienvenue dans ce cours complet qui retrace tout le développement du site BTS CIEL. Tu vas apprendre comment on a transformé un site HTML statique en une application web moderne avec Laravel, un dashboard admin sécurisé, et un déploiement Docker.

Le Contexte

Le Lycée Christophe Colomb possédait un site web pour la section BTS CIEL (Cybersécurité, Informatique et réseaux, Électronique). Ce site était entièrement en HTML/CSS statique, ce qui rendait les mises à jour difficiles pour les enseignants.

Les problèmes du site existant :

  • ❌ Pages HTML statiques difficiles à maintenir
  • ❌ Pas de système de gestion de contenu
  • ❌ Pas d'espace administrateur
  • ❌ Duplication de code (header/footer sur chaque page)
  • ❌ Déploiement manuel

Notre solution :

  • ✅ Migration vers Laravel (framework PHP moderne)
  • ✅ Dashboard administrateur complet
  • ✅ Système d'authentification sécurisé
  • ✅ Gestion des actualités et fichiers
  • ✅ Containerisation Docker

Technologies Utilisées

Laravel 11 PHP 8.3 MySQL 8.0 Docker HTML5 CSS3 jQuery

Ce que j'ai appris

  • Comment migrer un site statique vers Laravel
  • L'architecture MVC (Modèle-Vue-Contrôleur)
  • Créer un système d'authentification sécurisé
  • Développer un dashboard administrateur
  • Implémenter les opérations CRUD
  • Sécuriser une application web
  • Containeriser avec Docker

Migration vers Laravel

La première étape consistait à transformer les pages HTML statiques en pages dynamiques Laravel utilisant le moteur de templates Blade.

C'est quoi Laravel ?

Laravel est un framework PHP qui facilite le développement d'applications web. Il fournit des outils et une structure pour organiser ton code proprement.

Analogie simple

Imagine que tu construis une maison. Tu peux tout faire toi-même (couler le béton, poser les briques...) ou utiliser des éléments préfabriqués. Laravel, c'est comme utiliser des éléments préfabriqués : plus rapide, plus solide, plus propre !

Le Layout Principal

Le layout est le "squelette" de ton site. Il contient tout ce qui est commun à toutes les pages.

resources/views/layouts/template.blade.php 
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>@yield('title') - Lycée Christophe Colomb</title>
    
    <!-- CSS commun à toutes les pages -->
    <link rel="stylesheet" href="{{ asset('css/style.css') }}">
    <link rel="stylesheet" href="{{ asset('css/bootstrap.min.css') }}">
    
    @stack('styles') <!-- CSS spécifique à certaines pages -->
</head>
<body>
    <!-- HEADER (écrit UNE SEULE fois) -->
    <header>
        <nav>
            <a href="{{ route('home') }}">Accueil</a>
            <a href="{{ route('cours') }}">Cours</a>
            <a href="{{ route('projets') }}">Projets</a>
        </nav>
    </header>
    
    <!-- CONTENU DYNAMIQUE -->
    @yield('content')
    
    <!-- FOOTER (écrit UNE SEULE fois) -->
    <footer>
        <p>© 2024 Lycée Christophe Colomb - Développé par Fahel Brik</p>
    </footer>
    
    <!-- Scripts -->
    <script src="{{ asset('js/jquery.min.js') }}"></script>
    @stack('scripts')
</body>
</html>

Architecture MVC

MVC signifie Modèle-Vue-Contrôleur. C'est une façon d'organiser ton code en séparant les responsabilités. Laravel suit cette architecture.

Les 3 composants

┌─────────────────────────────────────────────────────────────────────────────┐
│                           ARCHITECTURE MVC                                   │
└─────────────────────────────────────────────────────────────────────────────┘

     UTILISATEUR
         │
         │ 1. Requête HTTP (ex: GET /cours)
         ▼
    ┌─────────────────────────────────────────────────────────────┐
    │                     ROUTES (web.php)                         │
    │     Route::get('/cours', [PublicController::class, 'cours'])│
    └─────────────────────────────────────────────────────────────┘
         │
         │ 2. Appelle le contrôleur
         ▼
    ┌─────────────────────────────────────────────────────────────┐
    │                  CONTROLLER (Contrôleur)                     │
    │                                                              │
    │   public function cours() {                                  │
    │       $cours = Cours::all();  // 3. Demande les données     │
    │       return view('cours', compact('cours')); // 5. Retourne│
    │   }                                                          │
    └─────────────────────────────────────────────────────────────┘
         │                                          │
         │ 3. Récupère les données                  │ 5. Envoie à la vue
         ▼                                          ▼
    ┌──────────────────────┐            ┌──────────────────────────┐
    │    MODEL (Modèle)    │            │      VIEW (Vue)          │
    │                      │            │                          │
    │ - Représente les     │            │ - Affiche les données    │
    │   données (BDD)      │            │ - Fichiers .blade.php    │
    │ - Eloquent ORM       │            │ - HTML + CSS + données   │
    │                      │            │                          │
    │   News.php           │            │   cours.blade.php        │
    │   File.php           │            │   projets.blade.php      │
    │   AdminUser.php      │            │                          │
    └──────────────────────┘            └──────────────────────────┘
         │                                          │
         │ 4. Renvoie les données                   │ 6. HTML final
         └──────────────────────────────────────────┘
                                                    │
                                                    ▼
                                             UTILISATEUR
                                          (voit la page web)
Composant Rôle Emplacement Laravel
Model (Modèle) Gère les données et la base de données app/Models/
View (Vue) Affiche les données (HTML) resources/views/
Controller (Contrôleur) Fait le lien entre Model et View app/Http/Controllers/

Les Routes

Les routes définissent quelle URL appelle quelle fonction. C'est un peu comme le "GPS" de l'application.

Route::get('/iris', [PublicController::class, 'iris'])->name('iris');

// Routes d'authentification
Route::prefix('admin')->group(function () {
    Route::get('/login', [AuthController::class, 'showLogin'])->name('admin.login');
    Route::post('/login', [AuthController::class, 'login']);
    Route::post('/logout', [AuthController::class, 'logout'])->name('admin.logout');

// Routes admin protégées (nécessitent d'être connecté)
Route::middleware(['admin.auth'])->prefix('admin')->group(function () {
    Route::get('/dashboard', [AdminController::class, 'dashboard'])->name('admin.dashboard');
    Route::get('/news', [AdminController::class, 'index'])->name('admin.news.index');
    // ...
});
Comprendre une route

Route::get('/cours', [PublicController::class, 'cours'])->name('cours');

• get : Méthode HTTP (get = afficher, post = envoyer des données)
• /cours : L'URL dans le navigateur
• PublicController::class : Le contrôleur qui gère cette route
• 'cours' : La méthode du contrôleur à appeler
• name('cours') : Un nom pour référencer cette route facilement

Le Contrôleur

Le contrôleur contient la logique de l'application. Il reçoit la requête, traite les données, et retourne une vue.

app/Http/Controllers/PublicController.php

    {
        return view('cours');
    }
    
    // Affiche la page iris (BTS CIEL)
    public function iris()
    {
        return view('iris');
    }
    
    // Affiche la page projets
    public function projets()
    {
        return view('projets');
    }
}

Le Modèle (Eloquent ORM)

Le modèle représente une table de la base de données. Eloquent permet de manipuler les données sans écrire de SQL.

app/Models/News.php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class News extends Model
{
    // La table associée (par défaut: "news")
    protected $table = 'news';
    
    // Les champs qu'on peut remplir
    protected $fillable = [
        'title',
        'content',
        'is_published',
        'published_at'
    ];
    
    // Conversion automatique des types
    protected $casts = [
        'is_published' => 'boolean',
        'published_at' => 'datetime'
    ];
}

Utilisation d'Eloquent :

Action SQL classique Eloquent
Tout récupérer SELECT * FROM news News::all()
Trouver par ID SELECT * FROM news WHERE id = 5 News::find(5)
Créer INSERT INTO news ... News::create([...])
Modifier UPDATE news SET ... WHERE id = 5 $news->update([...])
Supprimer DELETE FROM news WHERE id = 5 $news->delete()
Filtrer SELECT * FROM news WHERE is_published = 1 News::where('is_published', true)->get()

Résumé du Chapitre

  • MVC sépare les responsabilités : Modèle (données), Vue (affichage), Contrôleur (logique)
  • Les routes définissent quelle URL appelle quelle fonction
  • Le contrôleur traite la requête et retourne une vue
  • Eloquent ORM permet de manipuler la BDD sans SQL

Système d'Authentification

L'authentification permet de vérifier l'identité d'un utilisateur. On a créé un système de connexion sécurisé pour les administrateurs.

Le Contrôleur d'Authentification

app/Http/Controllers/AuthController.php 
<?php

namespace App\Http\Controllers;

use App\Models\AdminUser;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;

class AuthController extends Controller
{
    // Affiche le formulaire de connexion
    public function showLogin()
    {
        return view('admin.login');
    }
    
    // Traite la soumission du formulaire
    public function login(Request $request)
    {
        // 1. Valider les données reçues
        $request->validate([
            'username' => 'required',
            'password' => 'required'
        ]);
        
        // 2. Chercher l'utilisateur en BDD
        $admin = AdminUser::where('username', $request->username)->first();
        
        // 3. Vérifier si l'utilisateur existe ET si le mot de passe est correct
        if ($admin && Hash::check($request->password, $admin->password)) {
            // ✅ Connexion réussie !
            
            // Stocker l'ID admin dans la session
            session(['admin_id' => $admin->id]);
            session(['admin_username' => $admin->username]);
            session(['last_activity' => time()]); // Pour le timeout
            
            // Rediriger vers le dashboard
            return redirect()->route('admin.dashboard')
                             ->with('success', 'Bienvenue !');
        }
        
        // ❌ Échec de connexion
        return back()->withErrors(['login' => 'Identifiants incorrects']);
    }
    
    // Déconnexion
    public function logout()
    {
        // Supprimer les données de session
        session()->forget(['admin_id', 'admin_username', 'last_activity']);
        
        return redirect()->route('admin.login')
                         ->with('success', 'Vous êtes déconnecté');
    }
}
Sécurité : Hash::check()

On n'utilise JAMAIS $request->password == $admin->password !
Le mot de passe en BDD est haché (ex: $2y$12$xyz...).
Hash::check() compare le mot de passe saisi avec le hash stocké de manière sécurisée.

Important : @csrf

La directive @csrf est OBLIGATOIRE dans tous les formulaires POST.
Elle génère un token qui protège contre les attaques CSRF (Cross-Site Request Forgery).
Sans elle, Laravel refusera la soumission du formulaire !

Le Middleware de Protection

Un middleware est un "gardien" qui vérifie quelque chose avant de laisser passer la requête.

app/Http/Middleware/AdminAuth.php 
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class AdminAuth
{
    public function handle(Request $request, Closure $next)
    {
        // Vérifier si l'admin est connecté (session existe)
        if (!session('admin_id')) {
            // ❌ Pas connecté → Redirection vers login
            return redirect()->route('admin.login')
                             ->with('error', 'Veuillez vous connecter');
        }
        
        // ✅ Connecté → Laisser passer la requête
        return $next($request);
    }
}

Gestion des Actualités (CRUD)

CRUD signifie Create, Read, Update, Delete. Ce sont les 4 opérations de base pour gérer des données. On va les implémenter pour les actualités.

CREATE - Créer une actualité

L'utilisateur accède au formulaire

Route GET /admin/news/create → affiche le formulaire vide

L'utilisateur remplit et soumet

Le formulaire envoie les données en POST à /admin/news

Le contrôleur traite et enregistre

Validation des données → Création en BDD → Redirection

AdminController.php - Méthodes Create 
// Affiche le formulaire de création
public function create()
{
    return view('admin.news.create');
}

// Enregistre la nouvelle actualité
public function store(Request $request)
{
    // 1. Valider les données
    $request->validate([
        'title' => 'required|max:255',
        'content' => 'required',
    ]);
    
    // 2. Créer l'actualité en BDD
    News::create([
        'title' => $request->title,
        'content' => $request->content,
        'is_published' => $request->has('is_published'),
        'published_at' => now()
    ]);
    
    // 3. Rediriger avec message de succès
    return redirect()->route('admin.news.index')
                     ->with('success', 'Actualité créée avec succès !');
}

public function update(Request $request, News $news)
{
    // 1. Valider
    $request->validate([
        'title' => 'required|max:255',
        'content' => 'required',
    ]);
    
    // 2. Mettre à jour
    $news->update([
        'title' => $request->title,
        'content' => $request->content,
        'is_published' => $request->has('is_published')
    ]);
    
    // 3. Rediriger
    return redirect()->route('admin.news.index')
                     ->with('success', 'Actualité modifiée !');
}

Upload de Fichiers

FileManagerController.php - Upload 
public function upload(Request $request)
{
    // 1. Valider le fichier
    $request->validate([
        'file' => 'required|file|max:65536', // Max 64 Mo
        'category' => 'required'
    ]);
    
    // 2. Récupérer le fichier
    $file = $request->file('file');
    
    // 3. Générer un nom unique
    $filename = time() . '_' . $file->getClientOriginalName();
    
    // 4. Déplacer le fichier
    $path = $file->storeAs(
        'public/uploads/' . $request->category,
        $filename
    );
    
    // 5. Enregistrer en BDD
    File::create([
        'filename' => $filename,
        'original_name' => $file->getClientOriginalName(),
        'category' => $request->category,
        'path' => $path,
        'size' => $file->getSize(),
        'extension' => $file->getClientOriginalExtension(),
        'uploaded_by' => session('admin_id')
    ]);
    
    return back()->with('success', 'Fichier uploadé !');
}

Sécurité de l'Application

La sécurité est primordiale pour une application web. Voici les mesures qu'on a mises en place.

Hashage des Mots de Passe (bcrypt)

On ne stocke JAMAIS un mot de passe en clair. On le "hache" avec bcrypt. 

┌─────────────────────────────────────────────────────────────────────────────┐
│                        HASHAGE BCRYPT                                        │
└─────────────────────────────────────────────────────────────────────────────┘

  Mot de passe saisi: "admin123"
           │
           │  bcrypt()
           ▼
  Hash stocké: "$2y$12$0ZdDyzOzDnvgBDJBhzGPuOgRNCMBAMWQr/keo0vPH9j0Q1T.jiG3C"
  
  
  ⚠️ IMPORTANT:
  - Le hash est IRRÉVERSIBLE (on ne peut pas retrouver "admin123")
  - Chaque hash est UNIQUE (même mot de passe = hash différent)
  - Hash::check() compare sans révéler le mot de passe original
Création d'un admin avec mot de passe haché 
// ❌ JAMAIS ça !
AdminUser::create([
    'username' => 'admin',
    'password' => 'admin123'  // En clair = DANGER !
]);

// ✅ Toujours hasher !
AdminUser::create([
    'username' => 'admin',
    'password' => bcrypt('admin123')  // Haché = Sécurisé !
]);

Session Timeout (Déconnexion auto)

Après 10 minutes d'inactivité, l'administrateur est automatiquement déconnecté.

app/Http/Middleware/AdminSessionTimeout.php 
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class AdminSessionTimeout
{
    protected $timeout = 600; // 10 minutes en secondes
    
    public function handle(Request $request, Closure $next)
    {
        // Récupérer le timestamp de la dernière activité
        $lastActivity = session('last_activity');
        
        if ($lastActivity) {
            // Calculer le temps écoulé
            $elapsed = time() - $lastActivity;
            
            if ($elapsed > $this->timeout) {
                // ⏰ Timeout ! Déconnecter l'utilisateur
                session()->forget(['admin_id', 'admin_username', 'last_activity']);
                
                return redirect()->route('admin.login')
                    ->with('error', 'Session expirée après 10 minutes d\'inactivité');
            }
        }
        
        // Mettre à jour le timestamp
        session(['last_activity' => time()]);
        
        return $next($request);
    }
}

Protection CSRF

CSRF (Cross-Site Request Forgery) est une attaque où un site malveillant fait des requêtes à ton site en utilisant la session de l'utilisateur. 

┌─────────────────────────────────────────────────────────────────────────────┐
│                        ATTAQUE CSRF                                          │
└─────────────────────────────────────────────────────────────────────────────┘

  Sans protection CSRF:
  
  1. Admin connecté sur ton-site.com
  2. Admin visite site-malveillant.com
  3. Site malveillant contient:
     <form action="ton-site.com/admin/delete/5" method="POST">
       <input type="submit">
     </form>
  4. ❌ L'actualité #5 est supprimée !
  
  
  Avec protection CSRF (@csrf):
  
  1. Chaque formulaire a un token unique
  2. Laravel vérifie ce token à chaque requête POST
  3. Le site malveillant ne connaît pas le token
  4. ✅ La requête est rejetée !

Résumé Sécurité

  • bcrypt() pour hasher les mots de passe
  • Hash::check() pour vérifier les mots de passe
  • Session timeout après 10 min d'inactivité
  • @csrf obligatoire dans tous les formulaires POST
  • Middleware pour protéger les routes admin

Le Dockerfile

Le Dockerfile est la "recette" pour créer le container de l'application.

Dockerfile 
# Image de base: PHP 8.3 avec Apache

# Installer les extensions PHP nécessaires
RUN apt-get update && apt-get install -y \
    libpng-dev \
    libzip-dev \
    libonig-dev \
    && docker-php-ext-install pdo_mysql mbstring gd zip

# Activer le module Apache rewrite (pour Laravel)
RUN a2enmod rewrite

# Configurer Apache pour pointer vers /public
RUN sed -i 's|/var/www/html|/var/www/html/public|g' \
    /etc/apache2/sites-available/000-default.conf

# Installer Composer
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# Copier le projet
WORKDIR /var/www/html
COPY . .

# Installer les dépendances PHP
RUN composer install --no-dev --optimize-autoloader

# Permissions pour Laravel
RUN chown -R www-data:www-data /var/www/html/storage
RUN chmod -R 775 /var/www/html/storage

# Exposer le port 80
EXPOSE 80

Docker Compose

Docker Compose orchestre plusieurs containers ensemble.

services:
  # Container 1: Application Laravel
  app:
    build: .
    container_name: bts-ciel-app
    ports:
      - "80:80"  # localhost:80 → container:80
    volumes:
      - ./public/img:/var/www/html/public/img
    depends_on:
      db:
        condition: service_healthy
    
  # Container 2: Base de données MySQL
  db:
    image: mysql:8.0
    container_name: bts-ciel-db
    environment:
      MYSQL_ROOT_PASSWORD: root_secret
      MYSQL_DATABASE: bts_ciel
      MYSQL_USER: bts_user
      MYSQL_PASSWORD: bts_secret
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
  
  # Container 3: phpMyAdmin
  phpmyadmin:
    image: phpmyadmin
    container_name: bts-ciel-phpmyadmin
    ports:
      - "8080:80"
    environment:
      PMA_HOST: db

volumes:
  mysql_data:
